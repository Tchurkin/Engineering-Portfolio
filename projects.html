Ares - Thrust Vector Controlled Model Rocket

A fully self-stabilizing model rocket using thrust vector control (TVC). The system actively adjusts engine direction in real time to maintain stability during ascent.

  • Category: Controls / Embedded Systems / Aerospace
  • Status: Completed and iterated through flight tests

Problem: Model rockets are inherently unstable when actively controlled, especially with thrust vectoring. Stability during powered flight requires fast sensing, accurate state estimation, and robust control under changing dynamics.

My role: Designed and implemented the control system (initial PD, moving toward more advanced approaches), wrote embedded flight software on Teensy 4.1, integrated IMU + barometric sensors, designed the TVC actuator system, built the full hardware stack (PCB, wiring, structure), and ran simulation-to-real testing.

Core technologies: Embedded C/C++, MATLAB/Simulink, BNO055 IMU, DPS310 barometer, real-time control loops, PID/PD control.

Key challenges: Sensor noise and latency, actuator lag and mechanical limits, instability at motor burnout, and mapping simulation behavior to real flight.

Results: Achieved stable controlled ascent using TVC, demonstrated closed-loop control on a physically unstable system, and identified burnout instability as a key boundary for simple models.

Key insight: Controller performance degrades at regime transitions, motivating a shift toward model-based and observer-backed control methods.

Sisyphus - Propulsive Landing Model Rocket (In Progress)

A rocket platform designed for controlled, propulsive landing using thrust vector control. This extends Ares from stabilization into full trajectory control and landing.

  • Category: Advanced Controls / Autonomous Systems / Aerospace
  • Status: In progress

Problem: Propulsive landing requires real-time control of position, velocity, and orientation under tight constraints where small errors compound quickly.

My role: Designing full flight architecture across ascent, descent, and landing phases; developing landing-burn control strategy; integrating multi-sensor state estimation; implementing staged event logic (cutoff, re-ignition, landing legs, parachute backup); and expanding simulation to include descent dynamics.

Core technologies: Embedded C/C++, sensor fusion, real-time state estimation, advanced control methods moving beyond PD toward LQR/observer-based approaches, staged flight event logic.

Key challenges: Descent state estimation accuracy, landing burn timing, phase transitions, actuator/sensor delays, and safety redundancy.

Expected outcomes: Controlled descent and landing attempt, quantified trajectory tracking performance, and autonomous in-flight decision-making demonstration.

Why it matters: This moves from stabilizing attitude to controlling full trajectory, a major complexity jump aligned with real aerospace autonomy problems.

LinkedIn evidence: This project direction is driven by Ares post-flight transition analysis and control-boundary data.
